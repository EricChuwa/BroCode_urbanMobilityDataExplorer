## Setup & Installation

### 1. Clone the Repository

```bash
git clone https://github.com/EricChuwa/BroCode_urbanMobilityDataExplorer.git
cd BroCode_urbanMobilityDataExplorer
```

### 2. Configure Environment Variables

Copy the example env file and fill in your values:

```bash
cp .env.example .env
```

Your `.env` should contain:

```bash
DB_HOST=localhost
DB_PORT=5433
DB_USER=umde_user
DB_PASSWORD=umde_pass
DB_NAME=umde_db
PORT=3000
```

### 3. Start the Database

```bash
docker compose up -d
```

Verify it's running:

```bash
docker ps
# Should show umde-postgres running on port 5433
```

## рџ—„пёЏ Option A вЂ” Restore Full Database (Recommended)

Skip the pipeline entirely by restoring the pre-built database dump:

**Download the dump from Google Drive:**
> [umde_dump.pgdump](https://drive.google.com/file/d/17zWiBQG-VbxkjbpIcDStc0LaWVqR3KhB/view?usp=sharing)

Place it in `docs/` then restore:

```bash
docker exec -i umde-postgres pg_restore \
  -U umde_user \
  -d umde_db \
  -F c < docs/umde_dump.pgdump
```

Verify:

```bash
docker exec -it umde-postgres psql -U umde_user -d umde_db \
  -c "SELECT COUNT(*) FROM trips;"
# Should return 2,521,705
```

## рџ”„ Option B вЂ” Run the Full Pipeline

If you want to run the data pipeline from scratch:

### 1. Download the Dataset

Download the NYC Yellow Taxi parquet file and place it in `data/raw/tripData/`:

```
https://d37ci6vzurychx.cloudfront.net/trip-data/yellow_tripdata_2026-01.parquet
```

Also ensure `data/raw/taxi_zone_lookup.csv` is present.

### 2. Install Python Dependencies

```bash
pip install pandas pyarrow psycopg2-binary python-dotenv psutil --break-system-packages
```

### 3. Run the Pipeline

```bash
# Step 1 вЂ” Clean the data
python3 UMDE-BE/pipeline/loadClean_pipeline.py

# Step 2 вЂ” Engineer features
python3 UMDE-BE/pipeline/features.py

# Step 3 вЂ” Load into database
python3 UMDE-BE/pipeline/db_insert.py
```

## рџ“Ў API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/trips` | Trip records with filters |
| GET | `/api/zones` | All taxi zones |
| GET | `/api/zones/top-pickup` | Top N pickup zones (merge sort) |
| GET | `/api/summary/by-hour` | Trip counts by hour |
| GET | `/api/summary/by-borough` | Borough breakdown |
| GET | `/api/summary/by-time-of-day` | Time of day breakdown |

### Query Parameters for `/api/trips`

| Parameter | Type | Example |
|---|---|---|
| `borough` | string | `?borough=Manhattan` |
| `time_of_day` | string | `?time_of_day=morning` |
| `min_fare` | number | `?min_fare=10` |
| `max_fare` | number | `?max_fare=50` |
| `min_distance` | number | `?min_distance=1` |
| `max_distance` | number | `?max_distance=10` |
| `limit` | number | `?limit=100` |

## Custom Algorithm

A **merge sort** algorithm is manually implemented in `UMDE-BE/src/utils/algorithm.js` to rank taxi zones by trip volume without using JavaScript's built-in `.sort()`.

- **Time Complexity:** O(n log n)
- **Space Complexity:** O(n)
- **Used by:** `GET /api/zones/top-pickup`
